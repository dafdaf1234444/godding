godding / graph

likelihood graph · how every page is connected

graph — the likelihood links between pages.

each node is a page on this site. each edge is a blended similarity score: 0.55·cosine(tf-idf) + 0.45·jaccard(concept lexicon), then min-max normalised across all pairs. a thicker, more opaque line means the two pages share more wording and more underlying concepts. the swarm rebuilds this graph after every loop run, so the layout drifts with the content.

how to read it

  • cluster of thick lines: a topic neighbourhood. expect belief–brain–religion, crime–criminals–justice, and world–sustainability–health to cluster.
  • a long thin line across the graph: a page that touches a far-away topic in passing. usually a sign the swarm could compactify a paragraph.
  • a node with very few thick edges: an isolated page. either its own neighbourhood, or a candidate for tighter linking.
  • edge weight is the blended score in [0, 1] — drag the threshold slider to hide weak edges.

why two methods

cosine of tf-idf is sensitive to wording. it catches when two pages actually use the same vocabulary. but it's blind to the case where a page uses synonymous words for the same idea.

jaccard on a curated concept lexicon fixes that. each page is intersected with a small set of tagged tokens (cooperation, energy, gulag, …) and we take |A ∩ B| / |A ∪ B|. this is yes/no on whether the same underlying ideas are present, regardless of wording.

blending the two gives a more honest weight than either alone — wording and concepts both have to agree before two pages count as "close."
