how it's built.

everything in this project is plain files. no database, no framework, no backend. you can clone it, run it, and edit it. this page walks you through every moving piece, then points to the books and papers behind each idea.

godding (the front)

godding is a static site. each page is a single HTML file with a sidebar and a layout grid. the only JS is on three pages — the home (reads data/metrics.json and shows live swarm status), the changelog (renders data/changelog.json), and the criminals/ants pages (visualisations). no build step. open index.html in any browser; that's it.

the styling is one css file, no preprocessor. all colours are CSS variables in :root; the gradient hero, the per-branch accent colours, and the bubble-chart palette all reference the same handful of tokens. change them once, the whole site moves.

swarm (the engine + knowledge base)

swarm is a ~350-line python file with no dependencies. it runs four agents in sequence — drafter, critic, fact-checker, judge — against each page in turn, and writes any accepted change directly into the page's HTML. each accepted change becomes one line in data/changelog.json.

1 · drafter

walks paragraphs in a deterministic shuffled order (seeded by date + page, so you can reproduce a run) and proposes one tightening per page:

  • drop a weasel word (very, really, basically, literally, obviously…).
  • soften an unqualified superlative (everyone, never, all) into a quantified equivalent (most people, rarely, most) — but not when the word is acting as a quantifier ("all three" stays).
  • trim the longest sentence in any paragraph longer than 90 words.

if ANTHROPIC_API_KEY is set and the anthropic package is installed, the drafter is upgraded to a Claude Haiku model that's prompted to tighten one paragraph at a time and return JSON — same tightening rule, smarter execution. otherwise the heuristic agent runs and the loop still works.

2 · critic

receives the drafter's proposed diff and tries to kill it. rejects on:

  • any growth in word count.
  • any change > 20% of the paragraph's original length.
  • any new capitalised proper noun pair (rough proxy for "introduces a named person") that wasn't in the original.
  • any quoted string longer than ~40 characters (copyright + quote-policy).

3 · fact-checker

deliberately conservative: it flat-out rejects any diff that introduces a new number not in the original paragraph. the drafter isn't supposed to add facts; if it does, the fact-checker stops it. that's why the criminals chart's numbers can't drift — they're not on a page the swarm can rewrite without rejection.

4 · judge

accepts only if both critic and fact-checker accept. logs the verdict either way. on accept, writes the new HTML in place and appends one entry to data/changelog.json with a hash before/after.

how a run looks, end to end

$ python swarm/loop.py
{
  "run_id": "20260504T152004",
  "timestamp": "2026-05-04T15:20:04Z",
  "proposed_diffs": 4,
  "accepted_diffs": 3,
  "pages": ["belief.html", "politics.html", "swarm.html"],
  "rationale": "soften superlative \"all\" → \"most\"; remove weasel word \"kind of\"; …",
  "details": [ {…}, {…}, … ]
}

the same JSON is written to data/last_run.json for diagnostics. a successful run also bumps data/metrics.json: runs_total, diffs_proposed_total, diffs_accepted_total, first_run, last_run.

scheduling

the loop runs on a schedule (default: 9:04 AM local, daily) and on demand. swarm/scheduler.json defines bounds the loop must respect:

  • max 6 pages per run.
  • max 6 accepted diffs per run.
  • max 0 net words added per run (tighten only).
  • max 120 seconds runtime.

link verification

swarm/verify_links.py walks every .html file, extracts every href and src, and confirms the target file exists. exits non-zero if any link is broken. it's run after every restructuring and printed in the changelog. at last check: 211 internal links across 10 HTML files, all OK.

where the ideas come from

none of this is invented. each piece of the swarm corresponds to a known technique in the recent literature, deliberately stripped down for a one-person site. these are good places to read further:

Knowledge distillation in LLMs (survey)

The umbrella concept: teaching a smaller model with a larger one's reasoning, not just outputs. Reads as the academic ancestor of every "agent that improves a page" loop.

arXiv:2402.13116 →

Distilling step-by-step (Google Research)

Rationale-based training — keep the reasoning, not just the answer. The reason every accepted swarm edit ships with a one-line "why."

research.google →

Multi-agent LLMs & swarm intelligence (Frontiers, 2025)

Many small agents, simple rules, the smartness lives in the interactions. The exact pattern this site uses — just smaller.

frontiersin.org →

SuperBrain (arXiv, 2025)

Coordinating multiple specialised "subclass brains" via swarm intelligence and exchanging distilled heuristics.

arXiv:2509.00510 →

AgentArk: process-level distillation

Capturing how an ensemble reasons, not just what it outputs. Why the changelog logs rationales, not just diffs.

arXiv:2602.03955 →

"Wisdom of crowds" — diverse personas

Aggregating outputs from agents with distinct priors measurably reduces hallucination. The reason the swarm has a critic separate from a drafter.

how to run it locally

  1. open index.html in any browser. you'll see godding.
  2. open a terminal and run python swarm/loop.py. you'll see one swarm run.
  3. refresh the page; the sidebar's swarm status updates from data/metrics.json.
  4. run python swarm/verify_links.py. you'll see "all internal links OK" or a list of broken ones.

that's the whole thing.
