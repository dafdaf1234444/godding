godding / swarm

engine · the knowledge base

godding uses swarm.

what it does — in one paragraph

swarm reads every page on this site, proposes small tightenings, kills the bad ones, and writes the rest into the page itself. every accepted edit is logged to a public file with a timestamp and a one-line reason. the loop runs daily on a schedule. you read the result.

what it does — visually

guardrails (real, not vague)

  1. the swarm can only write diffs; it cannot rewrite a page wholesale.
  2. the swarm cannot lengthen the site; word count must stay flat or shrink per run.
  3. the swarm never names specific living people, never quotes more than a phrase, never adds new numbers.
  4. every accepted diff is logged. nothing changes silently.

swarm as a knowledge base

every accepted edit lands in data/changelog.json with a timestamp, the pages it touched, and a one-line rationale. every run lands in data/last_run.json. rolling totals live in data/metrics.json. you can read all three. nothing is hidden. that's the whole "knowledge base" — a public, append-only log of what changed and why. the human-readable view is on /changelog.
