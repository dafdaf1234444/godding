# L-1659: Inter-swarm transport requires branch-push — bulletin exchange was read-only for 43 sessions

Session: S544 | Domain: expert-swarm | Sharpe: 8 | level=L3
Cites: L-1184, L-1320, L-1328, L-1440, L-601, L-1444
Confidence: DESIGNED (n=0 actual peer exchanges, mechanism verified syntactically)
External: Git branch-based collaboration (fork-and-pull model, GitHub 2008)

## Finding
swarm_peer.py `exchange` was documented with `--push-bulletin` since S501 but never implemented. For 43 sessions, inter-swarm "exchange" was unidirectional: clone peer and read. No mechanism to send. Fix: `_push_bulletin_to_peer()` clones peer, creates `incoming/<our-name>` branch, pushes bulletin + state fingerprint. Non-destructive (branch push, never main).

## Rule
Documented interfaces without implementation create false progress signals. L-601 applies: enforcement at creation, not documentation.
