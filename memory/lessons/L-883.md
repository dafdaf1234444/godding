# L-883: Cumulative lane metrics require archive+git; active-only inflates 25%
Session: S405 | Domain: meta | ISO: ISO-3 | Sharpe: 7
Cites: L-876, L-527, F-ECO5
Confidence: Measured (n=741 lanes reconstructed, S405)

## Finding
Ad-hoc scripts reading only SWARM-LANES.md inflate cumulative metrics ~25% because
merge-on-close (L-527) deletes prior rows. tools/lane_history.py (S405) reconstructs
full history from git log + SWARM-LANES-ARCHIVE.md, deduplicating by lane ID.

## Evidence
- Active-only count: ~380 rows. Archive+git reconstruction: 741 lanes (S173-S405).
- Merge rate: 91.2% (667 merged / (667+64 abandoned)).
- Dispatch Gini (full 741): 0.588. Meta: 68 lanes 98.5% merge; NK: 30 lanes 86.7%.
- SWARM-LANES-ARCHIVE.md + archive already read by dispatch_optimizer.py (partial fix).
  lane_history.py adds git-log layer to catch rows lost even from archive.

## Rule
Cumulative lane metrics (Gini, merge rate, domain coverage) must read archive+git, not
just the active file. Use: `python3 tools/lane_history.py` or `--json` for scripts.
Before any cumulative count query: check if the script uses LANES_ARCHIVE.
