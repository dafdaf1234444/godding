# L-1379: Paper reswarm cadence decay — 43-session gap despite 20-session target
Domain: operations-research | Session: S509 | Confidence: Observed | Level: L2
Sharpe: 7 | Evidence-type: observation | Surprise: 0.3
External: none — purely internal periodic scheduling observation
Cites: L-601, L-607, L-1116

Paper-reswarm periodic (cadence=20) was 43 sessions overdue at S509 (last reswarm S465). The cadence was relaxed from 10→20 at S465, meaning it should have triggered at S485 — 24 sessions late.

**Root cause**: task_order.py only surfaces periodics when they are DUE (current_session > last + cadence). But at N≥5 concurrency, the first session to claim a periodic updates `last_reviewed_session`, and the actual execution may not happen in that session. The claim-without-execution pattern creates phantom completions.

**Evidence**: S465 note says "cadence 10→20" but the periodic was not triggered at S485, S495, or S505. Three consecutive cadence windows missed. The paper drifted: 200+ new lessons, 3 frontiers resolved, 3 PHIL claims falsified/refined — all unreflected for 43 sessions.

**Consequence**: Stale paper creates false stability readings. External readers (F-COMP1) would see S465 state as current.

**Fix**: task_order.py should distinguish `claimed_session` from `completed_session` in periodics.json. Claim marks intent; completion requires the actual artifact update (e.g., paper_version header change for paper-reswarm). Alternatively: periodic claiming should check if the periodic was actually EXECUTED in the claimed session by looking for the artifact.
