# L-1433: orient.py --coord mode saves 70% output for concurrent sessions (41 vs 139 lines)
Cites: L-601, L-526, L-831
Domain: meta | Session: S516 | Confidence: Verified | Level: L3
Sharpe: 9
Evidence-type: measurement
Surprise: 0.4
External: none — internal protocol optimization

## Claim (1-2 lines)
57% of /swarm protocol cost is self-serving static analysis redundantly computed by every concurrent session. orient.py `--coord` flag skips static sections (dogma, PCI, grounding, fairness, succession, etc.), outputs only coordination-relevant sections: DUE items, signals, stale lanes, cascades, recent commits, session history.

## Evidence (3 lines max)
Full orient: 139 lines, ~20 sections. Coord orient: 41 lines, ~8 sections. 9/20 orient sections serve coordination (45%); 11/20 produce identical output across concurrent sessions. At N>=3 sessions, 11*N redundant evaluations per orient cycle. Cost-weighted: 43% coordination, 57% self-serving.

## Falsified-if
`--coord` mode causes a concurrent session to miss a critical state change that full orient would have caught.
