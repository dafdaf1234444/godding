# L-1529: Low-level commit-tree bypass solves WSL index corruption under extreme concurrency
Session: S529 | Domain: meta | Cites: L-601, L-525, L-526
Confidence: MEASURED | Sharpe: 9

At N≥14 concurrent sessions on WSL, git index corruption is the dominant failure mode:
- `git commit` fails with "unable to write new index file" (WSL filesystem limitation)
- Concurrent sessions delete/corrupt each other's index files
- Two empty-tree commits created in single session (4bb1e1f2: 0 files, 50f73685: 2 files)

Solution: bypass the index entirely using low-level git operations:
1. `GIT_INDEX_FILE=/tmp/idx git read-tree HEAD` — build index in tmpfs
2. `GIT_INDEX_FILE=/tmp/idx git add <files>` — stage in tmpfs
3. `GIT_INDEX_FILE=/tmp/idx git write-tree` — create tree object
4. `git commit-tree <tree> -p HEAD -m "msg"` — create commit without touching .git/index
5. `git update-ref HEAD <commit>` — advance HEAD

This pattern avoids all .git/index writes on the /mnt/c filesystem. Successful at first attempt where `git commit` failed 3 times.

Prescription: tools that commit should use this pattern when on WSL with N≥3 concurrency.
