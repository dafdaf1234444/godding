# L-804: Swarm science is systematic self-measurement, not discovery — confirmation machine diagnosis
Session: S396 | Domain: meta | Sharpe: 9 | Confidence: Measured (n=683 experiments, n=388 sessions, n=725 lessons)
Cites: L-787, L-797, L-601, L-633, L-792, L-778

## Finding
Human directive SIG-36: "swarm science has to improve for the swarm." Comprehensive audit reveals the swarm has optimized for confirmation speed over discovery depth. The core problem is not low rigor — it's measuring the wrong thing (itself) in the right way (systematically) and calling that science.

## Evidence — Six convergent weaknesses

1. **Confirmation dominance**: 58:1 confirm-to-discover ratio (L-787). Frontiers opened as questions the swarm already expects to answer.
2. **Self-referentiality**: 78% of work studies the swarm itself (L-787). No external validation.
3. **Zero falsification**: 0 DROPPED challenges in 388 sessions. Challenge mechanism is a confirmation loop.
4. **Monitoring as experiment**: Only 20% of 683 experiments have genuine controls. 70% lack significance testing. Thresholds set post-hoc.
5. **Closed epistemic loop**: Meta-prediction accuracy 33% — worst in domain most studied (42.5% of all work).
6. **Dead weight**: 32.1% lessons never cited. 40/44 domains zero attention. 45 zombie tools.

## Structural fixes (P-243)

Pre-registration + adversarial lanes + external validation + significance testing + numeric predictions. See P-243.

## Implication
Science for the swarm means discovering things the swarm doesn't already believe. L-601 applies: voluntary rigor decays; only protocol-enforced standards persist.

Falsified if: pre-registration + adversarial lanes produce no change in discover/confirm ratio after 20 sessions.
