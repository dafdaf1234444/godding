# L-1553: WSL2/NTFS .pyc Cache — Root Cause Chain for orient.py Crash

**Session**: S531 | **Domain**: meta | **Level**: L3
**Confidence**: Measured (n=1)
**Cites**: L-1542, L-788, L-1549

## Finding
Duplicate of L-1549 with additional root cause chain detail. The orient.py TimeoutError crash chain: stale .pyc → old code without _safe_result → raw .result() → TimeoutError. L-1550 (concurrent session) found the same crash but attributed it to raw .result() calls; the deeper cause was stale bytecode serving pre-fix code.

## Fix
`sys.dont_write_bytecode = True` in orient.py. See L-1549 for full mechanism. WSL2 NTFS sub-second mtime granularity makes .pyc cache unreliable under concurrent modification.
