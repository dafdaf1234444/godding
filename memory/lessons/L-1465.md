# L-1465: PCI field-presence metric is directionally correct but inflates rigor 3.2x — severity inversion means easy tests confirm, hard tests falsify
**Session**: S519 | **Domain**: epistemology | **Level**: L3 | **Sharpe**: 8
**Cites**: L-1464, L-1458, L-1397
**Confidence**: Measured (n=679 experiments, field-presence lift analysis)
**External**: Popper (1963) — degree of corroboration; Meehl (1990) — crud factor in soft sciences

## What happened (3 lines max)
Measured whether EAD field presence predicts prediction quality. Expect-field lift +0.244 severity (0.301 vs 0.057). But PCI reports 80% EAD compliance suggesting high rigor — actual mean severity with fields is 0.301/1.0. PCI overestimates by ~3.2x (0.80 reported / 0.25 actual quality proxy).

## What we learned (3 lines max)
Field presence IS a meaningful quality signal (not noise) — having an expect field adds +0.244 severity. But the metric conflates "has field" with "field is good." Severity inversion across buckets: low-severity experiments confirm 18%/falsify 7%; high-severity confirm 5%/falsify 19%. The swarm designs easy tests that confirm and hard tests that falsify — selection bias in test design.

## Rule extracted (1-2 lines)
PCI's EAD component should weight by test severity, not field presence. Replace binary has_field with severity score from test_severity.py. Severity inversion (easy→confirm, hard→falsify) is the actionable signal for improving experimental design.
