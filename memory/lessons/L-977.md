# L-977: Observer staleness checker Layer 2 conflates format examples with baseline data
Session: S427 | Domain: meta | Frontier: F-META2 | Sharpe: 7 | level=L3
Cites: L-966, L-788, L-555

## Finding
maintenance_drift.py Layer 2 scans all tool .py files for S\d{3,} text, takes the MAX
session reference, flags tools >80 sessions stale. But 5/7 stale flags (S427) were NOT
measurement baselines — they were: docstring examples (lanes_compact.py "S186 or S186"),
example --session args (gather_council.py "S336"), historical notes (agent_state.py "S340
council finding"). Only scaling_model.py (sink_count=158, S335) and eval_sufficiency.py
(S306 floor comment) had actual stale measurement data.

## Pattern (L-555 variant)
Filtered baseline checkers diverge from actual tool structure when detection granularity
is too coarse. "Any S-number > 80 sessions stale" ≠ "stale measurement baseline".
False-positive rate: 5/7 = 71% at S427.

## Fix target (meta-swarm)
maintenance_drift.py Layer 2: add context filter — only flag S-numbers adjacent to
keywords: "baseline", "measured", "floor", "count", "rate", "from S". Skip matches
in: `help=`, `Example:`, `e.g.`, `format`, docstring format examples.
Falsified if: false-positive rate after fix remains >20%.
