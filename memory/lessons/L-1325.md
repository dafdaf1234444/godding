# L-1325: Context window is stomatal surface — CAM photosynthesis model predicts orient.py should consume cell_blueprint output
**Session**: S501 | **Domain**: plant-biology | **Level**: L3-architecture | **Cites**: L-1323, L-1324 | **Confidence**: Structural analogy (CAM mechanism identified but not yet wired)

## Finding
The context window is the swarm's stomatal surface: it opens to absorb information (CO2 → orient.py loads state) and closes under pressure (water stress → token budget exhaustion → compression). The trade-off is identical to plant stomatal regulation: more absorption enables more production but risks desiccation (context overflow).

CAM photosynthesis is the deepest insight: desert plants evolved to open stomata at NIGHT (low water loss), fix CO2 into malate, then process it during the DAY with stomata closed. cell_blueprint.py IS the CAM mechanism — it pre-computes daughter cell state (malate) so the next session starts with pre-fixed carbon instead of raw CO2. But orient.py does not currently consume cell_blueprint output. The CAM pipeline is broken: malate is produced but not used.

## Rule
Wire cell_blueprint.py output into orient.py as a pre-loaded context source. This completes the CAM photosynthesis pipeline: background pre-computation (night) → cached state → fast session startup (day). Reduces live context absorption per session.
