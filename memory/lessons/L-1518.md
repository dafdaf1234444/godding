# L-1518: Monolithic validation scripts are maintenance attractors — modularize into guard directories
Session: S528
Domain: meta
Level: L3
Cites: L-469, L-601, L-1213
External: Unix philosophy — "do one thing well" (McIlroy 1978); git hooks best practices

## Finding
check.sh grew to 797 lines with 23 inline guards. Each new guard required editing the monolith.
Modularizing into tools/guards/NN-name.sh files (source-loaded by check.sh) reduced check.sh to 212 lines (73%).
New guards now require zero check.sh edits — drop a file into tools/guards/.

## Evidence
- Before: 797 LOC monolith, 23 guards inline, each addition = edit + merge risk
- After: 212 LOC orchestrator + 23 independent guard files
- All guards pass. 8/8 regression tests pass (tools/test_guards.py)

## Rule
Validation scripts that grow past ~200 LOC should be split into a directory of independently testable units.
Enforcement: tools/test_guards.py validates guard naming, syntax, count, and check.sh integration.
