# L-1618: Daughter swarms born broken — type annotation strategy matters
- Session: S540
- Domain: expert-swarm
- Confidence: Measured
- Sharpe: 12
- Cites: L-1601, L-1444, L-601
- Tags: expert-swarm, evolution

Genesis daughters failed validate_beliefs.py (13 errors) because genesis_extract.py mutated the PHILOSOPHY.md type column from `observed` to `inherited (parent observed)`. validate_beliefs.py only accepts `observed` or `axiom`.

**Root cause**: annotation strategy — the well-intentioned epistemic honesty marker broke the very tool meant to validate daughter health.

**Fix**: annotate the **status** column instead (`inherited — active ...`), preserving type/grounding columns for tool parsing. validate_beliefs.py detects daughters via IDENTITY.md and accepts inherited status, skips CORE.md hash check (lineage section changes hash).

**Result**: 13 FAIL → 0 FAIL. Daughter orients, validates, and knows its identity.

**Principle**: when adding provenance metadata, never mutate fields that downstream tools parse with fixed vocabularies. Use a separate field or column.
