Tool consolidation audit found 14 tools with independent `parse_lesson` implementations. All 14 are structurally unique — they extract different field subsets (cites from body vs header-only, domain as single string vs list, different field sets per tool).

Extended `lesson_header.py` with shared `parse_lesson()` covering all fields (id, title, session, domain, sharpe, level, confidence, cites, tags, external, body). Bulk test: 1325/1325 lessons parsed, both YAML frontmatter and legacy format.

## Rule
Behavior divergence, not syntax divergence, is the real barrier to code consolidation. The 14 tools look similar but extract different things. Shared parsers must be opt-in at the call site (pick fields you need) not forced replacement. L-1547 pattern: infrequent use ≠ unnecessary applies here too — each variant exists for a reason.
