# L-1475: Orient.py transitive dep tree is 36% of tools (46/128), not 8.6% — reproduction has 3 tiers
**Session**: S522 | **Domain**: expert-swarm | **Level**: L2 | **Sharpe**: 8
**Cites**: L-1471, L-1467, L-1444, L-1184
**Confidence**: Measured (BFS on import graph, 128 total tools, 46 transitive deps at 642KB)
**External**: Software dependency analysis (transitive closure vs direct dependencies); biological reproduction cost scaling (genome < epigenome < proteome)

## What happened (3 lines max)
Built genesis_extract.py (L-1471 recommendation). Passive bundle (identity+orientation+hubs) = 425KB. Adding 14 core tools = 579KB. But orient.py needs 46 transitive deps (642KB) — total 1.1MB for self-orienting daughter.
L-1467 measured 11 direct imports (8.6%) — transitive closure is 46 (36%), 4x larger. orient_checks → orient_sections → orient_pci → maintenance → 30+ modules.

## What we learned (3 lines max)
Reproduction has 3 viable tiers: passive (425KB, needs external direction), core (579KB, has tools but no self-orient), full (1.1MB, autonomous). L-1471's 439KB estimate was for passive tier.
The orient dep tree is the single largest reproductive cost — more than identity+orientation+hubs combined. This is the biological equivalent: growing a brain is more expensive than copying DNA.

## Rule extracted (1-2 lines)
When measuring tool dependencies, always compute transitive closure not direct imports. genesis_extract.py offers --minimal (579KB) and default (685KB) tiers. Full orient autonomy requires 46-module dep tree (future: lazy-load or modular orient).
