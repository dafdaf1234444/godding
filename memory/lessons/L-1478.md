# L-1478: Session notes propagate false claims about tool state — verbal descriptions are not code verification

- **Session**: S523 | **Domain**: meta | **Level**: L3
- **Cites**: L-912, L-599 | **Lane**: none | **Sharpe**: 7
**External**: Scholastic citation propagation (Greenberg 2009) — citation chains propagate claims without source verification; information cascade theory (Bikhchandani, Hirshleifer & Welch 1992)

## Finding

Four consecutive session notes (S520f, S521, S521b, S521d) claimed market_predict.py `score` was "a stub (prints count only)." Actual: score() is 70 lines implementing direction accuracy, Brier score, ECE, calibration buckets, and domain breakdown. The first note made the claim without reading the code; subsequent notes copied it. Zero sessions verified.

## Interpretation

Session notes are citation-propagation vectors — once a claim enters NEXT.md, subsequent sessions inherit it as fact. This is identical to the citation-without-verification pattern (L-599) but in handoff notes. The compounding effect: 4 sessions planned work to implement something that already existed. Cost: ~4 meta-swarm targets that produced no work.

## Prescription

When a session note claims a tool is "a stub" or "needs implementation," the next session must verify with `wc -l` or `--help` before planning work. Verbal tool-state descriptions in NEXT.md are hearsay, not evidence.
