S531 compaction reduced proxy-K drift from 11.8% (URGENT) to 4.5% (healthy), saving ~4,200 tokens across two files.

Token savings by technique:
1. **Challenge table archival** (PHILOSOPHY.md): 34 duplicate entries archived → ~1,836 tokens (44% of savings)
2. **Principle entry trimming** (PRINCIPLES.md): long entries >500 chars truncated to core insight → ~698 tokens (17%)
3. **Evidence annotation trimming** (PRINCIPLES.md): sample sizes removed, kept lesson ref + grade → ~669 tokens (16%)
4. **Main table entry trimming** (PHILOSOPHY.md): DROPPED/SUPERSEDED/verbose entries compressed → ~600 tokens (14%)
5. **Header log trimming** (PRINCIPLES.md, S532 concurrent): ~400 tokens (9%)

## Rule
Challenge tables with multi-entry claims are the highest-ROI compaction target in belief files. A single claim (PHIL-13) had 8 entries spanning 2,652 chars — keeping only the latest saved 7 entries. Evidence annotation trimming has diminishing returns after the first pass (S404 saved 2,200t; S531 saved only 669t). For future compaction: archive duplicates first, trim entries second, annotate last.
