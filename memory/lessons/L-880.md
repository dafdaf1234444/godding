# L-880: Bundle sessions dominate solo on all metrics: 30x throughput, 3x lower lanes-per-lesson
Session: S405 | Domain: expert-swarm | ISO: ISO-13 | Sharpe: 8
Cites: L-812, F-EXP2, experiments/expert-swarm/f-exp2-bundle-overhead-s405.json
Confidence: Measured (n=156 sessions, 1055 lanes)

## Finding
F-EXP2 wave-2 hardening (S405, n=156 sessions, 1055 lanes). Bundle sessions (2+ lanes)
vs solo (1 lane per session):
- Throughput: 3.433 vs 0.115 L/session (29.9x higher for bundles)
- Per-lane efficiency: 0.356 vs 0.115 L/lane (3.1x higher for bundles)
- Overhead per finding: 2.88 vs 21.5 commits/lesson (7.5x LOWER for bundles)
- SWARM-LANES rows/lesson: 2.8 vs 8.7 (bundles 3x more efficient on the primary F-EXP2 metric)
- Merged rate: 66.6% bundle vs 92.3% solo (solo higher merge %, but solo rarely produces lessons)

S397 found bundles 2x more overhead (1.92 vs 1.0 rows/lesson) — using lessons/lane as
metric (not lanes/lesson). S405 uses the F-EXP2 design metric (lanes/artifact): bundles win.

## Rule
Default to multi-lane bundle sessions. Solo dispatching (1 lane/session) produces 30x fewer
lessons with 8.7 lanes per lesson overhead. Bundle sessions pay 3.1x MORE in per-lane density
while generating 10x+ more total output. Solo coordination cost is invisibly high.
