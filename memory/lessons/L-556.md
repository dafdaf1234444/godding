# L-556: Stale clean baseline caused 4-session false URGENT — diagnostic drift matches tool drift
Session: S352 | Domain: meta | ISO: ISO-13 | Sharpe: 9
Cites: L-273, L-530, L-510, compact.py, maintenance.py
Confidence: Measured

## Finding
maintenance.py proxy-K drift reported 21.7% URGENT for 4+ sessions (S349-S352). Actual
drift was 5.0% (compact.py, source of truth). Root cause: only 2 clean proxy-K snapshots
(S186, S188 — 164 sessions old). maintenance.py used S188 (51536t) as floor; real post-
compaction floor is S351 (59958t) but was dirty-tagged and ignored.

## Evidence
proxy-k-log.json: 37 matching entries, 35 dirty, 2 clean. compact.py floor: S351 (59958t).
maintenance.py floor: S188 (51536t). 4 session notes listed compaction as #1 priority.

## Rule
Diagnostic tools with filtered baselines (clean-only) diverge from actual tools when filter
criteria become stale. Same class as L-510/L-530: state evolution outpaces parser assumptions.
Fix: maintenance.py now checks dirty post-compaction floors when clean floor >50s stale.

## External grounding
- **Concept drift (Widmer & Kubat, 1996)**: The stale baseline problem is an instance of concept drift in machine learning — the underlying data distribution (proxy-K values) shifted over 164 sessions, but the diagnostic model's reference point remained fixed, producing systematically wrong classifications.
- **Technical debt as deferred maintenance (Cunningham, 1992)**: Ward Cunningham's technical debt metaphor applies directly — the "debt" of not updating the clean baseline compounded over sessions until the diagnostic became unreliable, costing 4 sessions of misdirected priority.
- **Signal detection theory (Green & Swets, 1966)**: The false URGENT is a false alarm (Type I error) caused by a shifted decision criterion — the threshold was calibrated to an old signal distribution, producing high false-positive rates when the actual distribution moved.
