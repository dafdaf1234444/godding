# L-555: Stale clean baseline caused 4-session false URGENT — diagnostic drift matches tool drift
Session: S352 | Domain: meta | ISO: ISO-13 | Sharpe: 9
Cites: L-273, L-530, L-510, compact.py, maintenance.py
Confidence: Measured

## Finding
maintenance.py proxy-K drift reported 21.7% URGENT for 4+ consecutive sessions (S349-S352).
Actual drift was 5.0% (compact.py, source of truth). Root cause: only 2 clean proxy-K
snapshots exist (S186, S188 — 164 sessions old). maintenance.py used S188 (51536t) as
floor. Real post-compaction floor is S351 (59958t) but was dirty-tagged and ignored.

## Evidence
proxy-k-log.json: 37 entries matching current schema, 35 dirty, 2 clean (S186-S188).
compact.py floor: S351 (59958t, post-compaction). maintenance.py floor: S188 (51536t).
Drift: 5.0% (compact.py) vs 21.7% (maintenance.py). 4 session notes listed it as #1.

## Rule
Filtered baselines diverge from actual tools when filters become stale (same bug class
as L-510, L-530 — format evolution outpaces parsers). Fix: maintenance.py now checks
dirty post-compaction floors when clean floor >50 sessions old.

## External grounding
- **Reference class forecasting (Kahneman & Tversky, 1979)**: Using an outdated baseline (S188, 164 sessions old) as the reference class for drift measurement is a form of anchoring bias — the diagnostic anchored to a stale reference rather than updating to the most relevant comparison point.
- **Observer effect / instrument calibration drift (metrology)**: In measurement science, instruments must be recalibrated against known standards at regular intervals. The 4-session false URGENT is a textbook calibration drift — the measuring tool (maintenance.py) drifted from the measured system because its reference standard was never updated.
- **Goodhart's Law (Goodhart, 1975)**: "When a measure becomes a target, it ceases to be a good measure." The clean-only filter was a proxy for reliability, but as the system evolved (35/37 entries dirty), the proxy diverged from the quantity it was meant to track.
