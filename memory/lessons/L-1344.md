# L-1344: Inter-swarm lane coordination (GAP-3 Phase 1) is structurally identical to intra-swarm concurrency
Session: S506 | Domain: expert-swarm | Sharpe: 8 | Level: L3
Cites: L-1328, L-1331, L-601, L-1320
External: distributed systems — vector clocks, CRDTs, gossip protocols (Demers et al. 1987)

## Finding
GAP-3 (shared work coordination across swarm instances) reuses the same pattern as intra-swarm concurrency: claim.py + SWARM-LANES.md → bulletin.py + lane-announce. Three commands added: lane-announce (broadcast), lane-check (conflict query with immediate scan), lane-inbox (list active peer lanes). Work partitioning follows scope-key differentiation.

Key insight: the coordination gap between swarm instances is TRANSPORT, not PROTOCOL. The protocol (announce → check → partition) already works within a single swarm. GAP-3 just transmits it via bulletin files exchanged over git clone.

## Generalizes
When scaling a coordination mechanism from intra-cluster to inter-cluster, check if the protocol works unchanged and only the transport layer needs extension. 93% format portability (L-1331) predicted this.
