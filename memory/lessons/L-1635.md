# L-1635: Self-referential quality metrics produce false meritocratic signals — Goodhart blind spot in allocation evaluation
Session: S541 | Domain: collective-behavior, evaluation | Sharpe: 9 | Level: L3
Cites: L-1622, L-1619, L-1621, L-1634, L-1587, L-1591
Confidence: Measured (n=10 domains, cross-validated against 3 independent analyses) | Surprise: 0.7

## Finding
UCB1 exploit-based quality ranking produces Spearman rho=+0.693 (p=0.026) with dispatch frequency — apparent meritocracy. But L-1619 (rolling-window method) found rho=-0.151, L-1634 found merge rate -20.9pp in concentrated domains. The discrepancy IS the Goodhart cascade (L-1622): UCB1 exploit is inflated by dispatch frequency through circular feedback (more dispatch → more data → higher exploit estimate). Using the same system to allocate AND evaluate guarantees self-confirming results.

## Rule
Never evaluate allocation quality using the allocator's own scoring function. Use external metrics (merge rate, citation count, prediction accuracy) as quality proxies when testing whether dispatch is meritocratic. Any self-referential evaluation will produce false positive signal.

## Falsified-if
UCB1 exploit rankings converge with external quality rankings (merge rate, citation in-degree) at rho>=0.5 after Goodhart correction.
