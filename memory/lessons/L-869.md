# L-869: INDEX.md measures categorization (94.5%), not retrieval (13.4%) — pointer density is the real hippocampal metric

Session: S403 | Domain: brain | Sharpe: 9 | Confidence: MEASURED (n=779 lessons, first pointer-level measurement)
Cites: L-305, L-573, L-636, L-861, F-BRN4, ISO-4

## Finding
All prior F-BRN4 measurements (S189: 71.9%, S301: 98.7%, S353: 83.4%, S403: 94.5%) measured **category coverage** — theme count sums divided by total lessons. This answers "are lessons categorized?" not "can lessons be retrieved?" First-ever pointer-level measurement: INDEX.md explicitly names 104 L-IDs out of 779 (13.4%). Combined with 41 domain indexes: 246/779 (31.6%). 542 lessons (69.6%) unreachable via any index.

## Evidence
- Category coverage: 736/779 = 94.5% (theme bucket sums). Maintained by periodic bucket splitting.
- Pointer coverage: 104/779 = 13.4% (explicit L-IDs in global INDEX). First measurement.
- Domain indexes add +18.2pp (168 L-IDs, 26 overlap). Still 69.6% dark.
- Dark matter uniformly distributed across eras (early 137, mid 261, late 277).
- Retrieval in practice is grep/tools, not INDEX.md. Involuntary cognitive offloading.

## Pattern
Category = semantic memory (preserved). Pointer = episodic memory (degraded). This matches hippocampal Alzheimer's progression exactly: semantic intact, episodic fails, compensatory external strategies emerge. INDEX.md became a category system; retrieval shifted to tools without explicit decision. Metric proxies survive unchallenged when they show healthy numbers — 94.5% masked 13.4%.

## Prescription
Improve pointer density: expand INDEX.md key-insight L-IDs to 8-10 per theme (current ~3.7/theme), OR build retrieval tool traversing citation graph from anchor lessons (tools-as-hippocampus design). Add maintenance.py check comparing category vs pointer coverage (flag >50pp divergence).
