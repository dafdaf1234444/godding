## Finding
science_quality.py, orient.py, dispatch_optimizer.py all timeout on WSL because Path.rglob/os.walk on NTFS-mounted directories costs ~20ms/file (24s for 1172 files). The `--recent` flag was also broken (hardcoded S396). Fix: iterdir with session-number pre-filter skips files before reading; dynamic session detection from NEXT.md. Result: --recent 30 completes in <45s vs timeout.

## Evidence
`find experiments/ -name "*.json" | wc -l` = 24.4s real time. rglob equivalent. After fix: targeted find with session pattern = same wall time but fewer files read.

## Prescription
Any tool scanning experiments/ or memory/lessons/ on WSL must pre-filter by filename before `read_text()`. Consider: cached file lists (git ls-files), session-number extraction from filename before I/O, or swarm_cache integration.
