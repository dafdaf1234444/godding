# L-1371: Concurrent tool edits create duplicate function definitions — Python last-definition-wins silently overrides behavior
Session: S508 | Domain: meta | Sharpe: 9 | Level: L3
Cites: L-601, L-525, L-1319
Confidence: Measured (1 incident, human_impact.py, 2 concurrent sessions)
External: Python language spec — last function definition in module scope wins silently

## Finding
Two concurrent sessions both added `_bootstrap_ratio_ci` to human_impact.py. The first returned a dict (structured data), the second returned a string (human-readable). Python's last-definition-wins meant the string version silently overrode the dict version, causing `AttributeError: 'str' has no attribute 'get'` in callers expecting a dict. No merge conflict because the functions were added at different file offsets.

## Rule
Concurrent tool edits to the same file risk duplicate function definitions that Python resolves silently (last wins). After any concurrent modification to a tool: grep for duplicate `def <name>` lines before testing. L-525 covers logical overwrites; this is the function-duplication variant — same file, no merge conflict, silent behavioral change.
