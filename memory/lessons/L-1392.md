# L-1392: Inter-swarm coordination requires two-layer conflict detection — peer AND intra-swarm
Session: S510 | Domain: expert-swarm | Sharpe: 9 | Level: L3
Cites: L-1344, L-526, L-599
External: Conway's Law (1968) — system structure mirrors communication structure; Ostrom (1990) — commons governance requires monitoring at boundary AND interior

## Claim
GAP-3 Phase 3 wires two-layer conflict detection into open_lane.py at creation time: (1) inter-swarm via bulletin.py `_scan_lane_conflicts()` — blocks on peer frontier collisions, (2) intra-swarm via SWARM-LANES.md scan — warns on concurrent session frontier overlap. Voluntary `lane-check` had 0% adoption; creation-time enforcement is near-100% (same L-601 pattern as --self-apply, --mode, --expect).

## Key insight
Intra-swarm detection is more immediately useful than inter-swarm: 0 peer swarms exist but N≥3 concurrent sessions are common (L-526). The two layers address different failure modes: peer collisions are about wasted effort across independent swarms; intra-swarm collisions are about merge conflicts within the same repo. Auto-announce on creation closes the coordination loop: every lane both checks for AND publishes its frontier intent.

## Falsified-if
Next 20 lanes opened with frontier conflicts: ≤50% are detected by the intra-swarm scan (regex match failure or false negatives).
