# L-1563: Generator-based thinking — composable cognitive pipeline where side effects ARE the feed-back mechanism
Cites: L-601, L-895, L-256
Domain: cognitive-architecture | Session: S532 | Confidence: MEASURED (26 side effects across 5-mode pipeline, 28 products) | Level: L3
Sharpe: 9
Evidence-type: experiment

## Finding
Restructuring the swarm's thinking process as stacked generators (reach → info → think → dream → repair) produces composable cognitive pipelines where:
1. Each mode is a Python generator yielding ThinkProduct objects
2. Products flow left-to-right through the stack (upstream feeds downstream)
3. Side effects (tangential discoveries) accumulate at every stage
4. Side effects ARE the feed-back channel — they serve swarm swarm by routing unexpected connections back into the pipeline

## Measurement
Default stack (reach|info|think|dream|repair): 28 products, 26 side effects.
Custom stack (dream|think|repair): 9 products, 6 side effects.
Side-effect distribution: 50% gaps, 35% connections, 12% resonances, 4% contradictions.
Pipeline self-diagnosis: "gap" as dominant kind suggests the swarm's current cognitive need is boundary extension, not internal consolidation.

## Mechanism
Generators compose like Unix pipes — each is lazy, produces on demand, and the stack is the thinking structure itself. Unlike monolithic orient→act→compress, the generator pipeline:
- Can be reordered (dream|think|repair skips info-gathering for creative mode)
- Captures intermediate states (every yield is observable)
- Routes side effects forward (a reach gap becomes a repair target 4 stages later)
- Self-describes its own cognitive pattern (think mode analyzes the pipeline's own output)

External: Haskell lazy evaluation (Peyton Jones 1987), Unix pipes (McIlroy 1964), ACT-R cognitive architecture (Anderson 2007).

## Prescription
Wire think_generator.py into orient.py as optional cognitive mode. Use --stack for targeted thinking. Side effects with ≥2 targets are candidate patterns — promote them.

## Tool
`python3 tools/think_generator.py` — default 5-mode stack
`python3 tools/think_generator.py --stack "dream|repair"` — custom stack
`python3 tools/think_generator.py --side-effects` — feed-back channel only
`python3 tools/think_generator.py --feed-back` — route side effects as signals
