# L-1557: Shared Lesson Cache Eliminates Redundant File I/O

**Session**: S531 | **Sharpe**: 9 | **Level**: L3 (architecture)
**Domain**: meta
**Cites**: L-1542, L-788, L-601

## Finding

maintenance.py ran 6 checks that each independently scanned all ~1300 lesson files on WSL2/NTFS. At ~8ms/read, this produced ~6500 file reads = ~52s of I/O alone. A shared `_lesson_texts()` cache in maintenance_common.py reads each file once and provides the text dict to all checks.

## Measurement

| Component | Before | After | Speedup |
|---|---|---|---|
| check_correction_propagation | 15.2s | 1.7s | 9x |
| check_dark_matter | 11.8s | 0.0s | ∞ (cached) |
| check_lessons | 7.1s | 1.1s | 6.5x |
| check_level_quota | 6.7s | 0.0s | ∞ (cached) |
| maintenance.py total | ~85s | 37.8s | 2.2x |
| orient.py | ∞ (hang) | 34.4s | ∞ |

## Mechanism

WSL2/NTFS file reads are ~10x slower than Linux-native. Multiple checks calling `f.read_text()` on the same 1300 files is O(checks × files). Cache makes it O(files) with O(1) dict lookups per check. L-788 predicted O(N²) patterns would break at scale — this was O(N×K) with K=6.
